PSI meeting October 2002

Protein-protein interaction agenda

This is a preliminary, open agenda. Please send me any additions/changes.
  1. Overall objective: What is the potential benefit of a common standard for protein-protein interaction data?
    Aim: Define use cases.
  2. Overview of related/similar efforts.
  3. Which data should be represented?
    1. Scope of the standard, granularity of data representation.
    2. data elements to be stored
    Aim: Outline data structure.
  4. Which existing systems/controlled vocabularies/ontologies can be used, where do we need additional systems?
    Aim: List of existing components to be used, list of required components, ideally in order of urgency.
  5. Representation of proposed data format: UML/XML?
  6. Data exchange:
    1. no coordination,
    2. federated database with data exchange,
    3. federated database with common query interface?
    4. how to avoid redundancy?
    5. by flat/XML files
    6. Define availability levels:
      1. free
      2. free to end user (no redistribution)
      3. only "detail" access via tools
      4. private
    7. Data ownership, update policy.
  7. Submission requirement by publishers: Current status.
  8. Specialisation/coordination between databases for human curation?
  9. Synchronised effort to create reference sets?
  10. Quality control: accept only published data?
