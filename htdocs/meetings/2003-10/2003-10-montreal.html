Meeting Report from the

2nd Annual HUPO congress, Montreal

9th October 2003

 

The need for common standards which will allow the interchange, comparison and public domain deposition of published proteomic data is a recognised need within the scientific community. To this end, the Proteomic Standards Initiative (PSI) of the Human Proteomic organisation (HUPO) has been working to develop such standards in collaboration with major database providers, instrumentation and software providers and other standardisation efforts in related fields. The PSI contribution to the second annual HUPO congress was divided into two parts. A full session at the conference, chaired by Rolf Apweiler and Weimin Zhu, was dedicated to describing the progress made to date in developing standards and interchange formats in the fields of protein-protein interactions (PPI) and mass spectrometry, whilst also highlighting areas where such efforts have yet to be initiated, for example in the storage of information from 2-D electophoresis gels. Secondly, the PSI held an open meeting at which the mass spectrometry XML interchange schema was successfully demonstrated and the further development of the PSI-MI standard discussed.

 

Conference Session

The conference session heard a brief introductory talk by Dr Rolf Apweiler (EBI) in which he described the history of the Proteomic Standards Initiative and summarised the progress made to date in each area. He also touched upon a new area of interest for the PSI, that of enzymology, and described a recent start-up meeting held in Germany.  Weimin Zhu (EBI) further described the development of the PSI-MS XML standard interchange format, which was then put into the context of the wider efforts to standardise the field of mass spectrometry by Dr Randall Julian (Eli Lilly). Sandra Orchard (EBI) announced the release of Level 1 of the PSI-MI interchange format, this announcement has also been submitted for publication and it is hoped that the paper will be in press in the very near future. Akihisa Kenmochi (Nec Corporation, Tsukuba, Japan) described an XML based proteomics-oriented markup language designed to accelerate collaboration among researchers, Veerasamy Ravichandran (National Institute of Standards and Technology, Gaithersburg, United States) spoke on the need for data standards for two-dimensional gel electrophoresis and Sandhya Xirasagas (Science Applications International Corporation, Germantown, MD, United States) discussed the CEBS object model for systems biology (CEBS MAGE Sys BioOM). Finally, Chris Taylor (EBI) talked about the PSI data integration model, MIAPE, which will be based upon the PEDRo model developed by the University of Manchester [1]. It is planned to develop different sub-domains which will allow the model to handle specific data types, for example 2-D electrophoresis gels or HPLC. The model will support the various PSI-interchange formats and also a number of ontologies which will be developed, in some cases in collaboration with the MGED consortium [2], and then collated for use by research workers.

 

Workshop Session

The meeting started with a show of hands, to decide whether to split into separate sessions to discuss PPI/mass spectrometry issues or to run the workshop as a single meeting. It transpired that very few people were interested in only a single aspect of proteomics data and the decision was taken to continue as a single meeting.

The mass spectrometry session was chaired by Weimin Zhu and opened by Kai Runte and Randall Julian undertaking a live demonstration on the PSI-MS XML data interchange format, alongside tools which will allow the user to

  • Convert from mass spectrometry text formats to PSI-MS XML format
  • View and browse stored in PSI-MS XML format
  • Use PSI-MS XML format as a search engine input

 

The standard is being developed in conjunction with the American Society for Tests and Measures (ASTM) which is developing a broader based XML-format for analytical information exchange [2]. The PSI-MS schema is flexible enough to handle a diversity of experiments with a full range of experimental descriptors whilst still remaining compliant with the ASTM model.

 

Concerns were raised as to the practicality of storing large amounts of data in XML format and the length of time it would take to access data stored sequentially. It was emphasised that this is an interchange format, not a storage medium, and that users should transfer the data into an appropriate repository. It is also not anticipated that vendors will replace their own proprietary binary file format with PSI-MS XML but rather that they would align with the XML export format and supply this as an option for users. The response of the vendors, to date, has been excellent and it is hoped that this will continue into the future.

 

Future plans include working to develop more converters capable of supporting peak list formats other than data (.DTA) files, to continue to implement an open source search engine from the Institute for Systems Biology (ISB) capable of taking the standard peaklist XML document as an input file and to develop a standardized output format from search engines. Both hardware and search engine vendors will be further encouraged to support the newly developed standards, and, in common with other PSI efforts, users will be encouraged to participate in the development of controlled vocabularies. All developments will be made public on the SourceForge website http://psidev.sourceforge.net/ms/

 

The PPI session was co-chaired by David Sherman (University of Bordeaux) and Jérôme Wojcik (Hybrigenics).  A number of issues arose from the previous presentation of the Level 1 interchange format but, as with the PSI-MS XML format, many of these were issues that will ultimately be addressed by the database into which the data is deposited. Questions were raised as to the mechanism by which new terms will be added to the already existing controlled vocabularies which describe interaction type, participant detection method etc. The GO model was outlined as a potential methodology which could be utilised and it was recognised that a firm decision needs to be made on this at the next PSI PPI meeting and the process then rapidly put in place and advertised to the user community. Finally an appeal was made for help in user evaluation of the Level 1 schema and assistance in developing Level 2, in the further development of tools to process the data when in this format and in expanding the current controlled vocabularies, as need arises.

 

The meeting closed with a final appeal for user participation and an invitation to all interested parties to sign onto the PSI mailing lists to be found on the PSI website (http://psidev.sourceforge.net/).
